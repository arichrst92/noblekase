# Dua Bahasa (ID / EN) ŌĆö Noblekase

Dibuat oleh: PT Solusi Inovasi Bangsa (https://ide.asia)

## Ringkasan

| Hal | Nilai |
|---|---|
| Bahasa default | Bahasa Indonesia (`id`) ŌĆö **tanpa** prefix URL |
| Bahasa kedua | Inggris (`en`) ŌĆö prefix `/en` |
| Contoh | `/produk` (ID) Ōåö `/en/produk` (EN) |
| Tombol ganti bahasa | Navbar kanan, tampil di desktop maupun mobile |

`/id/produk` sengaja dialihkan permanen (308) ke `/produk`, supaya tiap halaman
hanya punya satu URL kanonik.

## Alur teknis

```
URL publik          middleware.ts              route yang dirender
/produk       ŌåÆ     rewrite ke /id/produk  ŌåÆ   app/(frontend)/[locale]/produk
/en/produk    ŌåÆ     diteruskan             ŌåÆ   app/(frontend)/[locale]/produk
/id/produk    ŌåÆ     redirect 308 ke /produk
```

Rewrite (bukan redirect) dipakai untuk bahasa default agar URL Indonesia tetap
bersih dan semua tautan lama tetap hidup.

Segmen `[locale]` diteruskan ke:

1. **Query CMS** ŌĆö `payload.find({ locale })`, sehingga field `localized: true`
   mengembalikan terjemahannya.
2. **Komponen UI** ŌĆö prop `locale` untuk teks antarmuka dan prefix tautan.
3. **Metadata** ŌĆö `<html lang>`, `og:locale`, canonical, dan hreflang.

## Dua sumber teks

Teks di situs datang dari dua tempat, dan penting membedakannya:

| Sumber | Isi | Cara mengubah |
|---|---|---|
| **CMS** (Payload) | Judul produk, artikel, hero, halaman, FAQ ŌĆö semua yang dikelola editor | Buka dokumen di admin, ganti pemilih bahasa di kanan atas field, isi versi EN |
| **Kamus kode** (`src/lib/i18n.ts`) | Label antarmuka: "Cari", "Lihat semua", "Urutkan", pesan error, aria-label | Perlu perubahan kode |

Kamus kode juga menjadi **fallback** ketika field CMS masih kosong, jadi halaman
tidak pernah tampil dengan lubang kosong.

## Untuk editor: menerjemahkan konten

1. Buka dokumen di admin (produk, artikel, halaman, dll).
2. Di kanan atas ada pemilih bahasa ŌĆö pindah ke **English**.
3. Isi field yang ingin diterjemahkan, lalu simpan.

Field yang dibiarkan kosong di EN otomatis jatuh ke versi Indonesia
(`fallback: true` di `payload.config.ts`), jadi terjemahan boleh dicicil.

**Slug tidak diterjemahkan.** Satu produk memakai alamat yang sama di kedua
bahasa (`/produk/detail/charger-gan-65w` dan `/en/produk/detail/charger-gan-65w`).
Ini disengaja: tautan tetap stabil, dan Google lebih mudah memasangkan kedua
versi sebagai terjemahan.

## SEO

- Setiap halaman memuat `alternates.languages` (hreflang) berisi kedua bahasa
  plus `x-default` ke versi Indonesia.
- `canonical` selalu menunjuk URL bahasa halaman itu sendiri.
- `sitemap.xml` mendaftarkan kedua bahasa untuk setiap URL, lengkap dengan
  hreflang per entri.
- Halaman hasil pencarian (`/cari`) sengaja `noindex, follow` ŌĆö tiap kata kunci
  menghasilkan URL baru, dan dengan dua bahasa jumlahnya berlipat.
- Saat dokumen dipublish, cache kedua bahasa disegarkan dan kedua URL diajukan
  ke Google Indexing API (lihat `src/lib/publishHooks.ts`).

## Menambah kunci teks baru (developer)

Di `src/lib/i18n.ts`, tambahkan kunci ke objek `id` **dan** `en`:

```ts
const id = {
  "product.newLabel": "Label baru",
  // ŌĆ”
};

const en: Record<TranslationKey, string> = {
  "product.newLabel": "New label",
  // ŌĆ”
};
```

Objek `en` diketik `Record<TranslationKey, string>`, jadi kunci yang lupa
diterjemahkan **menggagalkan build** ŌĆö bukan diam-diam muncul sebagai teks
Indonesia di halaman Inggris.

Pemakaian di komponen:

```tsx
const tr = translator(locale);
<span>{tr("product.newLabel")}</span>
```

Dengan sisipan nilai:

```ts
tr("products.showingCount", { shown: 12, total: 40 });
```

## Menambah bahasa ketiga

1. Tambahkan kode locale di `locales` (`src/lib/i18n.ts`) dan di
   `localization.locales` (`src/payload.config.ts`) ŌĆö keduanya harus sama.
2. Lengkapi `htmlLang`, `ogLocale`, `localeLabel`.
3. Tambahkan objek kamus baru bertipe `Record<TranslationKey, string>`.

Middleware, sitemap, hreflang, dan tombol pemilih bahasa membaca daftar
`locales`, jadi semuanya ikut menyesuaikan tanpa perubahan lain.

## Yang perlu dicek saat testing

- [ ] Tombol ID/EN berpindah tanpa keluar dari halaman yang sedang dibuka
- [ ] Filter & kata kunci pencarian tidak hilang saat ganti bahasa
- [ ] `/id/produk` teralihkan ke `/produk`
- [ ] URL ngawur seperti `/en/xyz` menampilkan 404 berbahasa Inggris
- [ ] `/admin` dan `/api/*` tidak terpengaruh
- [ ] Tautan di dalam isi artikel (rich text) tetap berada di `/en`
- [ ] `view-source` halaman EN: `<html lang="en-US">` dan tag hreflang lengkap
